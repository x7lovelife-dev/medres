# 使用说明

## 在 Codex 中使用

在本仓库根目录开启 Codex 后，可直接提出类似任务：

```text
使用 pharma-research-agent 分析 CLDN18.2 在胃癌中的全球竞争格局，并生成 Markdown 报告。
```

```text
使用 pharma-research-agent 比较两个药物在某适应症中的临床证据。
```

```text
使用 pharma-research-agent 分析某药物相关重点不良事件信号。
```

临床对比问题如果范围较宽，智能体会先做范围收窄。例如：

```text
调研药物A联合标准治疗 vs 药物B联合标准治疗在某疾病中的临床证据。
```

智能体应先确认或自行声明范围，例如：

```text
本次默认范围：指定疾病场景、指定治疗线、指定人群/亚型、指定给药方式下，比较药物A + 标准治疗与药物B + 标准治疗。优先检索直接头对头研究；若无成熟结果，再使用同治疗线、相似人群和相似方案的强间接证据，并把早期、单臂、不同人群或不同方案研究放入背景证据。
```

## MCP 部署

当前仓库已完成 `pharma_research` MCP server。先安装依赖：

```powershell
pip install -r requirements.txt
```

再启动服务：

```powershell
python mcp-server/server.py
```

把 `.codex/config.example.toml` 复制为 `.codex/config.toml`，或合并到用户级 `~/.codex/config.toml`：

```toml
[mcp_servers.pharma_research]
command = "python"
args = ["mcp-server/server.py"]
cwd = "C:/Users/ALT147/Documents/多角色分析自动流程拆解"
startup_timeout_sec = 20
tool_timeout_sec = 120
```

配置后重启 Codex 或新开线程。

## 当前能力边界

当前仓库已完成 V0 工作流包和 MCP 工具原型：

- Codex Skill 可发现的目录结构。
- 多角色职责卡。
- 三类报告模板。
- 多源数据地图。
- 证据质量规则。
- 问题收窄与范围界定规则。
- `search_literature`：PubMed、Europe PMC、Crossref。
- `search_clinical_trials`：ClinicalTrials.gov API v2。
- `search_drug_labels`：openFDA Drug Label。
- `search_adverse_events`：openFDA FAERS 聚合。
- `search_patents`：PatentsView 结构化检索原型；公共源不可用时返回 Google Patents、Espacenet、PatentsView 检索入口。

运行调研时，如果某个外部源超时或不可达，MCP 会返回 `warnings`，报告应保留该检索缺口。
