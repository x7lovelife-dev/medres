# 使用说明

## 在 Codex 中使用

在本仓库根目录开启 Codex 后，可直接提出类似任务：

```text
使用 pharma-research-agent 分析 CLDN18.2 在胃癌中的全球竞争格局，并生成 Markdown 报告。
```

```text
使用 pharma-research-agent 比较 nivolumab 和 pembrolizumab 在胃癌中的临床证据。
```

```text
使用 pharma-research-agent 分析 osimertinib 相关间质性肺病的不良事件信号。
```

## MCP 部署

当前仓库已完成 `search_literature` 的本地 Python 逻辑和 MCP 入口。先安装依赖：

```powershell
pip install -r requirements.txt
```

再启动服务：

```powershell
python mcp-server/server.py
```

把 `.codex/config.example.toml` 复制为 `.codex/config.toml`，或合并到用户级 `~/.codex/config.toml`：

```toml
[mcp_servers.pharma_research]
command = "python"
args = ["mcp-server/server.py"]
cwd = "C:/Users/ALT147/Documents/多角色分析自动流程拆解"
startup_timeout_sec = 20
tool_timeout_sec = 120
```

配置后重启 Codex 或新开线程。

## 当前能力边界

当前仓库已完成 V0 工作流包和第一个 MCP 工具原型：

- Codex Skill 可发现的目录结构。
- 多角色职责卡。
- 三类报告模板。
- 多源数据地图。
- 证据质量规则。
- MCP 工具契约规划。
- search_literature 已实现 PubMed、Europe PMC、Crossref 的查询聚合和 DOI/标题去重。

其余 MCP 工具仍未实现。运行调研时，如果没有可调用 MCP，Codex 应使用当前可用的联网/API 能力，或在报告中标注“计划调用但当前未安装 MCP 工具”。
