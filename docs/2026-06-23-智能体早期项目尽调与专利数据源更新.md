# 智能体早期项目尽调与专利数据源更新

更新时间：2026-06-23
仓库：x7lovelife-dev/medres

## 更新目标

本次更新把 `pharma-research-agent` 从临床证据和安全性分析扩展为可支持早期药物项目第三方尽调的多角色工作流。新的设计保留单一 Skill 入口，但把早期项目、非临床证据、制剂可行性和项目质疑规则拆成独立 reference 文件，避免每次任务都加载全部角色和模板。

## 主要变化

### 1. 主 Skill 改为按任务路由

`.agents/skills/pharma-research-agent/SKILL.md` 新增以下研究模式：

- `target-validation`：靶点-疾病证据、人类证据、表达、通路和转化支持。
- `modality-feasibility`：制剂或技术路线是否适配靶点和组织场景。
- `nonclinical-evidence-review`：体外、体内、PK/PD、毒理和模型可转化性评估。
- `early-asset-diligence`：早期药物项目第三方可行性、缺陷和失败路径评估。

临床、安全性和靶点竞争格局任务继续读取通用参考文件；早期项目任务读取 `early-asset-*` 专用参考文件。

### 2. 新增早期项目专用 reference 文件

新增文件：

- `early-asset-role-cards.md`
- `early-asset-data-source-map.md`
- `early-asset-evidence-rules.md`
- `early-asset-report-template.md`

这些文件分别定义早期项目角色分工、数据源优先级、证据等级和报告结构。核心要求是每个项目结论必须同时列出支持证据、反对证据、关键不确定性、失败路径和建议补充实验。

### 3. 增加项目质疑和第三方尽调机制

早期项目团队新增或强调以下角色：

- 靶点生物学 Agent
- 制剂/技术路线 Agent
- 非临床药效 Agent
- PK/PD Agent
- 毒理风险 Agent
- 竞品与失败先例 Agent
- 专利与差异化 Agent
- 项目质疑 Agent

其中项目质疑 Agent 负责独立审查正向叙事，提出不可验证假设、同靶点失败、模型不可转化、暴露不足、组织表达风险、专利受限等可能导致项目失败的路径。

### 4. 专利数据源优先级调整

Google Patents 网页仍只作为低频核验和链接入口，不作为批量抓取后端。

Google Patents Public Datasets / BigQuery 被纳入结构化专利数据源，用于：

- 专利 landscape
- 批量统计
- 申请人趋势
- CPC 和技术方向分析
- 专利族和 publication number 追踪

使用 BigQuery 前必须限定查询范围，并记录预计扫描量、SQL 条件和查询日期。默认禁止未限定全表扫描或 `SELECT *`。

## 调用逻辑

推荐调用流程：

1. 用户提出研究问题。
2. Skill 先判断任务类型。
3. 临床任务加载通用/临床参考文件。
4. 早期项目任务加载 early-asset 专用参考文件。
5. 信息采集员先查结构化数据源，再查权威会议、监管机构、公司公告和必要网页线索。
6. 各角色输出结构化中间结论。
7. 项目质疑 Agent 审查支持证据和反对证据。
8. 报告撰写 Agent 生成 Markdown 报告。
9. 溯源质控 Agent 检查来源 ID、证据等级、过度外推和缺失证据。

## 当前能力边界

- 本次更新主要是 Skill 和参考文档层面的工作流升级。
- MCP 代码仍保持当前实现：文献、临床试验、药品标签、不良事件和 PatentsView 专利检索原型。
- Google Patents Public Datasets / BigQuery 已进入数据源规划和优先级规则，但尚未实现为 MCP 工具。
- 后续实现 BigQuery 接入时，应先提供 SQL 预览和预计扫描量，再由用户确认执行。
