# 医药分析多角色智能体仓库约定

本仓库用于沉淀可在 Codex 中长期运行的医药分析多角色智能体工作流包。

## 工作方式

- 优先使用仓库内 `.agents/skills/pharma-research-agent` Skill 执行医药深度调研任务。
- 方案、需求、流程和使用说明放在 `docs/`。
- 报告默认输出到 `reports/`，文件名使用 `YYYY-MM-DD-topic-research.md`。
- MCP 工具接口规划放在 `mcp-server/`，在真实服务实现前不要把接口描述成已可调用。

## 输出要求

- 每条关键结论必须关联来源标识，或明确标注为“综合判断”或“待补充证据”。
- 文献、临床试验、不良事件和药品标签数据需要保留来源、检索日期和可追溯 ID。
- 面向内部分析使用，文档中不默认加入公众医疗免责声明。
