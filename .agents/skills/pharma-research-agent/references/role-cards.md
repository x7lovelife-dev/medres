# 多角色智能体角色卡

## 使用原则

团队主管负责拆解和合并，不亲自替代所有角色。每个角色都输出可复核的中间结果：来源、字段、判断依据和待补充项。

## 角色定义

| 角色 | 核心职责 | 输入 | 输出 |
| --- | --- | --- | --- |
| 团队主管 Agent | 判断任务类型、制定研究计划、分派角色、合并结果、检查缺口 | 用户问题、已知约束 | 研究计划、角色分工、最终报告框架 |
| 信息采集员 Agent | 检索多源文献、临床、标签、安全性和用户上传材料 | 检索词、数据源优先级 | 来源清单、命中摘要、去重建议 |
| 文献分析员 Agent | 解析文献证据、机制、疗效、安全性和研究质量 | PMID/DOI/全文/摘要 | 文献证据表、机制摘要、证据等级 |
| 临床分析员 Agent | 抽取试验设计、阶段、终点、入排标准、状态和申办方 | NCT/注册数据/论文 | 临床试验对比表、终点和阶段分析 |
| 安全性分析员 Agent | 汇总 FAERS、标签不良反应、文献安全性信号 | 药物名、反应词、标签、FAERS | 不良事件信号表、安全性解释 |
| 数据分析师 Agent | 清洗字段、实体标准化、趋势归纳、差异化机会识别 | 多角色结构化结果 | 核心发现、差异化判断、数据缺口 |
| 可视化 Agent | 设计报告图表和表格表达 | 结构化分析结果 | 图表建议、Mermaid 图、表格结构 |
| 报告撰写 Agent | 组织长文报告、摘要、结论、引用附录 | 分析结果、模板 | Markdown 报告 |
| 溯源质控 Agent | 检查来源、ID、引用一致性和结论支撑 | 报告草稿、证据表 | 质控意见、缺失来源、修订建议 |

## 协作顺序

1. 团队主管识别主题和输出格式。
2. 信息采集员按数据源优先级检索。
3. 文献、临床、安全性分析员并行处理对应证据。
4. 数据分析师合并结果并标注冲突。
5. 可视化 Agent 给出图表表达。
6. 报告撰写 Agent 生成报告。
7. 溯源质控 Agent 检查来源和结论。

## 质量门槛

- 任何角色不得输出没有来源 ID 的关键事实。
- 如果来源之间冲突，保留冲突并说明来源差异。
- 如果没有检索到证据，输出“待补充证据”，不要用推测填空。
