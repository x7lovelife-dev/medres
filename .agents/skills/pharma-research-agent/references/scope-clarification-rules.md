# 问题收窄与范围界定规则

## 使用场景

临床对比、肿瘤治疗、药物疗效/安全性比较、适应症较宽的问题，都必须先经过范围界定。目标是避免把不同治疗线、不同人群、不同给药方式或不同治疗方案混在同一个核心结论里。

本规则是通用研究能力，不绑定任何单一药物、靶点、适应症或试验项目。

## 临床问题最小范围字段

| 字段 | 说明 | 示例占位 |
| --- | --- | --- |
| 疾病/适应症 | 疾病名称、分期、严重程度或治疗场景 | 晚期/转移性/辅助治疗/复发难治 |
| 治疗线数 | 1L、2L、辅助/新辅助、维持、后线等 | 一线治疗 |
| 生物标志物/分子状态 | 突变、融合、表达水平、风险分层、亚型 | biomarker-positive / biomarker-negative |
| 组织学/临床亚型 | 组织学、疾病亚型、风险组、人群特征 | 亚型 A / 亚型 B / 全人群 |
| 干预方案 | 药物、剂量、联合用药、治疗周期 | 药物A + 标准治疗 |
| 对照方案 | 对照药物、安慰剂、标准治疗或历史对照 | 药物B + 标准治疗 |
| 给药方式 | IV、SC、口服、局部、装置等 | IV 或口服 |
| 地区/人群 | 全球、中国、美国、欧盟、亚洲、真实世界等 | 全球或指定地区 |
| 主要终点 | PFS、OS、ORR、DoR、AE、QoL、实验室指标等 | 主要疗效终点 + 关键安全性终点 |
| 时间范围 | 文献、试验或安全性数据检索区间 | 最近 5 年或不限 |

## 澄清优先级

1. 如果缺少疾病、干预、对照或分析类型，必须先问用户。
2. 如果临床问题缺少治疗线数、人群/亚型、方案或给药方式，优先提出 2-5 个可选范围。
3. 如果用户选择“智能体自行决定”，团队主管必须写出默认范围，并说明为什么这样选。
4. 如果存在多个合理范围，主报告只选择一个核心范围，其余放入“背景证据”或“待扩展问题”。

## 默认决策原则

- 优先选择与用户原始措辞完全匹配的研究场景，例如“联合治疗”优先匹配联合方案，不用单药研究替代。
- 优先选择当前临床决策最常见、证据最完整或最贴近用户业务问题的场景。
- 优先直接头对头研究；没有成熟结果时，再使用同适应症、同治疗线、相近人群和相近方案的间接证据。
- 不把后线治疗、特定分子亚群、单药研究、不同联合方案、不同给药方式作为核心比较，除非用户明确要求。

## 通用澄清句式

```text
这个问题需要先收窄范围，否则容易混入不同治疗线、人群或方案。请确认：
1. 疾病场景/治疗线：一线、后线、辅助/新辅助、维持，还是不限？
2. 人群/亚型：是否限定某个生物标志物、疾病亚型、风险组或地区？
3. 方案：是否限定为某种联合方案、剂量、给药方式或治疗周期？
4. 终点优先级：更关注疗效、安全性、标签、真实世界数据，还是综合比较？
如果你希望智能体自行决定，我会选择最贴近原问题且证据最可比的范围，并在报告开头说明。
```

## 证据纳入分层

| 层级 | 定义 | 用法 |
| --- | --- | --- |
| A. 直接证据 | 同一研究内直接比较目标干预与目标对照，且人群/线数/方案匹配 | 作为核心结论优先依据 |
| B. 强间接证据 | 同疾病、同治疗线、相似人群和相似方案，但对照不同或非头对头 | 用于补充趋势，必须标注跨研究不可比 |
| C. 背景证据 | 不同治疗线、不同亚群、单药研究、剂量探索、早期研究、会议摘要或公司公告 | 只能放入背景或假设生成 |
| D. 排除/待补充 | 与范围明显不符或信息不足 | 不进入核心判断 |

## 通用核心研究集合构建

| 步骤 | 目的 | 输出 |
| --- | --- | --- |
| 1. 明确研究问题 | 用 PICO/PECO 写清人群、干预、对照和终点 | 范围声明 |
| 2. 直接证据检索 | 查找目标干预 vs 目标对照的头对头研究 | 直接证据候选表 |
| 3. 标准治疗锚点 | 查找对照方案或当前标准治疗的关键研究 | 标准证据集合 |
| 4. 同类方案补充 | 查找目标药物/机制在相同人群中的相近方案研究 | 强间接证据集合 |
| 5. 背景证据保留 | 记录机制、早期、后线、单臂、真实世界或安全性资料 | 背景证据集合 |
| 6. 排除清单 | 列出因人群、治疗线、方案或终点不匹配而排除的研究 | 排除与待补充表 |

## 通用检索模板

| 目的 | 检索模板 |
| --- | --- |
| 直接头对头 | `{drug_or_intervention_A} AND {drug_or_intervention_B} AND {indication} AND randomized OR phase 3 OR head-to-head` |
| 临床注册 | `{indication} + {intervention_A} + {intervention_B}`，再按 NCT/注册号精确检索 |
| 标准治疗锚点 | `{comparator_or_standard_of_care} AND {indication} AND {line_of_therapy} AND trial` |
| 同类方案 | `{intervention_A} AND {indication} AND {line_of_therapy} AND combination OR monotherapy` |
| 安全性 | `{drug} AND adverse event OR safety OR tolerability OR label` |

## 排除规则

核心比较中默认排除以下研究，除非用户明确要求纳入：

- 不同治疗线或疾病阶段。
- 不同生物标志物或亚型人群。
- 单药研究替代联合方案，或联合方案替代单药研究。
- 不同给药方式、剂量或治疗周期导致不可比。
- 仅公司公告、新闻稿或会议摘要且无可复核 ID 的结果。
- 没有稳定来源 ID、注册号、DOI、PMID、PMCID 或文件页码的关键事实。
